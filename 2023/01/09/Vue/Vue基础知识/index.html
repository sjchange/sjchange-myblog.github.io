数据驱动视图

数据的变化会驱动视图自动更新

双向数据绑定

再网页中 form表单负责采集数据 ajax 负责提交数据
js数据的变化 会被自动渲染到页面上
页面上的表单采集的数据发生变化时 会被vue自动获取 并更新到js数据中
数据驱动视图和双向数据绑定的底层原理时mvvm(Mode数据源 View视图 ViewModel就是vue的实例)

内容渲染指令

v-text 指令的缺点 会覆盖元素内部原有的内容

插值表达式

实际开发中用的最多 只是内容的占位符 不会覆盖原有的内容

v-html 指令的作用 可以把带有标签的字符串 渲染成真正的HTML内容

属性绑定指令

注意：插值表达式只能用在元素的内容节点中，不能用在元素的属性节点中！
v-bind 为元素的属性 动态绑定 指令可以简写成 :
如果绑定内容需要进行动态拼接，则字符串的外面应该包裹单引号
v-on 绑定事件指令 用来为dom元素绑定事件监听 简写 @

在绑定事件处理函数的时候 可以使用小括号传参

vue 提供了 内置对象 名字叫$event 他就是原生的事件对象e

v-model 双向数据绑定指令 用来辅助开发者在不操作dom的前端下 快速获取表单数据

条件渲染指令

v-show 的原理是 动态为元素添加 或移除 display:none 样式 来实现元素的显示和隐藏

v-if 的原理是 每次动态创建或移除元素 实现元素的显示和移除

过滤器

要定义到filters节点下 本质是一个函数
在过滤器函数中 一定要有 return值
在过滤器的形参中 就可以获取到 ‘管道符’ 前面待处理的那个值
如果全局过滤器和私有过滤器名字一致时 此时就按’就近原则’

watch侦听器

针对数据的变化 做特定的操作
方法格式的侦听器
缺点: 无法在刚进去页面的时候 自动触发
如果侦听的是一个对象 如果对象的属性发生了变化 不会触发 侦听器
对象格式的侦听器、
好处: 可以通过 immediate选项 让侦听器自动触发
深度侦听
可以通过deep选项 开启 深度监听 对象属性变化 也会触发 侦听器

计算属性

通过一系列运算之后 最终得到一个属性值
定义要被定义方法
在使用计应属性时 当普通的属性使用即可
实现代码的复用
只要计算性中依赖的数据源的变化 则计算属性会自动重新求值

axios

axios是一个专注 网络数据请求的数据库

在终端下运行 vue create 项目

vue的语法

(1)、数据绑定
    1)、{{ 变量 }}  ->将数据绑定到html,文本的原样输出绑定
        {{ js的表达式 }} ->输出表达式的运算结果
    2)、v-html="变量"  ->将数据保留html标签格式输出到html页面
        <div v-html="变量"></div>   将变量中的内容保留标签格式，插入到div标签中
    3)、v-text="变量" ->将数据原样输出到html页面中
        <div v-text="变量"></div> 将数据原样输出到html页面中，插入到div标签中
    4)、v-if="表达式" ->条件判断指令 
        <div v-if="表达式"></div> 如果表达式成立，则输出显示div的内容
        
        v-else-if="表达式" ->多分支条件语句
        
        v-else  ->与就近的v-if匹配使用，后面不带条件
    5)、v-show="表达式" ->如果表达式成立，则显示相应的内容    如果表达式不成立，则显示相应的内容，但是是通过样式display:none 进行内容隐藏(html代码是存在页面上)
    6)、v-once ->只渲染一次，后续值再变，则html显示的结果不变

指令 v-for ->循环指令(循环数据，生成列表)

首先：先有数据数组或者对象
(1)、结构1:
      <div v-for="value in object"></div>
      注释：
        object  ->需要循环的数据对象数组
        in ->固定语法 
        value ->逐次循环得到的元素值或者对象的属性值
(2)、结构2：
     <div v-for="(val,key) in Object">
     </div>
     注释：
        object  ->需要循环的数据对象数组
        in ->固定语法 
        val ->逐次循环得到的元素值或者对象的属性值
        key ->数组元素的下标或者是对象的属性的名称
(3)、结构3：
     <div v-for="(val,key,index) in Object">
     </div>
     注释：
        object  ->需要循环的数据对象数组
        in ->固定语法 
        val ->逐次循环得到的元素值或者对象的属性值
        key -> 数组元素的下标或者是对象的属性的名称
        index -> 属性的索引   如果循环的是数组，则无索引
(4)、结构4：
    <div v-for="value in n"></div>
     注释：
        n  ->需要循环的整数，从1开始循环，一直到N
        in ->固定语法 
        value ->逐次循环得到的数

弹性布局：

(1)、父级元素
    1)、display:flex  ->讲布局设为弹性布局
    2)、justify-content ->控制子级元素在主轴上的对齐方式
        flex-start[默认值]  ->主轴的开始方向对齐
        flex-end  ->主轴的结束方向对齐
        center  ->->主轴的中间方向对齐
        space-between ->主轴两端对齐，并且子元素与父元素无缝隙
        space-around ->主轴两端对齐，并且子元素与父元素有缝隙(子元素之间距离的1/2)
    3)、flex-direction：控制主轴与交叉轴的方向
         row[默认值] ->主轴方向 从左到右边
         row-reverse ：主轴方向 从右到左
         column ->主轴方向 从上到下
         column-reverse; ->主轴方向 从下到上
    4)、flex-wrap ->子元素是否换行 
         nowrap[默认值] ->不换行
         wrap  ->换行
         wrap-reverse ->换行并且反向换行
    5)、align-items ->子元素在交叉轴的对齐方式(当子级元素只有一行式有效)
        flex-start ->在开始位置对齐
        flex-end  ->在结束位置对齐
        center  ->中间对齐
        baseline -> 控制子级元素的第一行文本的底端对齐。
        stretch[默认值] -> 如果子级元素没有设置高度，则平铺满父级元素
    6)、align-content：子元素在交叉轴的对齐方式(当子级元素多行有效)     
        flex-start ->开始方向对齐
        flex-end  ->结束方向对齐
        center  ->居中对齐
        space-between ->两端对齐，无缝隙
        space-around ->两端对齐，有缝隙 缝隙=子级元素的一半
        stretch[默认值] ->子级元素均匀分布，并且最后一行子级与父级有缝隙 缝隙=子级的间距
(2)、子级元素
     1)、order 属性定义子元素的排列顺序。数值越小，排列越靠前，默认为0
     2)、flex-grow 属性定义子元素的放大比例，默认为0，即如果存在剩余空间，也不放大
     3)、flex-shrink属性定义了子元素的缩小比例，默认为1，即如果空间不足，该项目将缩小。
     4)、flex-basis ->设置子级元素的固定宽度，不会受放大或者缩小影响
     5)、align-self ->单独控制子级元素在交叉轴的对齐方式

props 如果是一个对象，对象的属性名称，就是接收父级传输过来的属性名称，属性值，指的是属性存储的值的类型。
